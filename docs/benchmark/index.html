Evidence contract

How Zero-Jank Scroll evaluates scroll architecture

This project does not label an interaction “smooth” based only on code appearance. It separates static source evidence from runtime browser evidence and reports what was actually measured.

Runtime procedure

  1. Record browser, version, operating system, viewport, zoom, input method, and throttling.
  2. Capture a performance trace around the interaction.
  3. Inspect long tasks, scripting, style/layout, paint, and layout shifts.
  4. For framework projects, compare render counts before and after.
  5. Repeat slow, fast, reverse, click-to-step, resize, and reduced-motion tests.

What this project will not claim

  • A universal FPS number without a recorded trace.
  • That transforms and opacity guarantee zero jank.
  • That IntersectionObserver is correct for every continuous effect.
  • That GSAP, Motion, or native CSS is always the winner.

Use the included source audit to find leads, then verify them in a real browser. Static findings are not runtime proof.
