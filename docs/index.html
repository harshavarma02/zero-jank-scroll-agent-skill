Architecture before animation

One interaction class. One deliberate choice.

The skill first classifies the interaction as discrete, continuous, or cinematic. It then chooses platform primitives or an animation library based on what the interface actually needs.

Discrete activation

Real step elements, CSS sticky positioning, one IntersectionObserver, and state changes only when identity changes.

Continuous mapping

CSS scroll timelines where support permits, or a single controlled progress primitive outside framework render state.

Cinematic choreography

GSAP ScrollTrigger when precise scrubbed timelines, coordinated scenes, and lifecycle refresh behavior are central.

Matched comparison

The common generated version versus the maintainable version.

Before: fragile

  • Hardcoded 400vh track
  • Global scroll listener
  • Repeated layout reads and DOM writes
  • Percentage state math
  • Manual scroll coordinates
  • Paint-heavy transitions

After: fixed

  • Semantic steps create the timeline
  • CSS owns sticky positioning
  • Top-aligned IntersectionObserver trigger
  • State changes only at real boundaries
  • scrollIntoView on real controls
  • Reduced-motion and short-viewport fallbacks
